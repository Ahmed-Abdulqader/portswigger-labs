## Objective  
This lab contains a SQL injection vulnerability in the product category filter. The application displays the results of the database query, making it vulnerable to `UNION`-based attacks.

To solve the lab, perform a SQL injection attack using the `UNION` operator that returns an additional row containing `NULL` values. This confirms the number of columns returned by the original query.

---

## Background  
A **`UNION` SQL injection attack** allows an attacker to retrieve data from other database tables by appending a new `SELECT` query to the original one. For this to work, two key conditions must be met:
1. The injected query must return the **same number of columns** as the original.
2. The data types in each column must be **compatible**.

This vulnerability occurs when user input (e.g., URL parameters) is **concatenated directly into SQL queries** without using parameterized queries or input validation.

---
### My Approach

1. I selected the **"Pets"** category and observed the URL:
   ```
   /filter?category=Pets
   ```

2. I tested for SQL injection by appending a single quote:
   ```
   /filter?category=Pets'
   ```
   This caused a database error, confirming the input was being injected into a SQL query.

3. I began testing with `UNION SELECT NULL` payloads, incrementing the number of `NULL`s until the application returned a valid response with an extra row.

4. After several attempts, I found that **three `NULL` values** matched the column count.

---

#### Testing Breakdown

- **Initial injection test** — Start with the following to force a column mismatch error (proof the vulnerability is exploitable):
   ```sql
   category=Gifts' UNION SELECT NULL--
   ```
   This caused an error indicating the column counts don't match.

- **Increment `NULL` values** — Incrementally add `NULL` until the application shows an extra empty row:

| Payload | Result |
|---|---|
| `UNION SELECT NULL` | ❌ Error — 1 column vs 3 columns |
| `UNION SELECT NULL,NULL` | ❌ Error — 2 columns vs 3 columns |
| `UNION SELECT NULL,NULL,NULL` | ✅ **Success!** — 3 columns match |

> [!NOTE] 
> The `%23--%2b` at the end (i.e., `-- +`) **neutralizes syntax** like trailing quotes or clauses (e.g., `AND released='1`). The `+%23` can also be `--+` or `;%23` depending on the application's query structure.

## Payload Used  
```
https://<LAB-ID>.web-security-academy.net/filter?category=Pets%27+UNION+SELECT+NULL,NULL,NULL--+
```

**Decoded for clarity:**
```
category=Pets' UNION SELECT NULL,NULL,NULL--+
```

---

## Why It Worked  
The original query likely looked like this:
```sql
SELECT * FROM products WHERE category = 'Pets' AND released = 1
```

After injection, it became:
```sql
SELECT * FROM products WHERE category = 'Pets' UNION SELECT NULL,NULL,NULL--+' AND released = 1
```

Breakdown:
- `'` — closes the string in the `WHERE` clause.
- `UNION SELECT NULL,NULL,NULL` — appends a second query with **three columns**, matching the original `SELECT *` (which returns multiple columns, but in this case, exactly three are expected).
- `--` — starts a SQL comment, neutralizing the rest of the original query (`AND released = 1'`).
- `+` — URL-encoded space, ensures clean parsing.

Since the number of columns matched and `NULL` is compatible with most data types, the database executed the query and returned an additional row of `NULL` values — solving the lab.

---

## How to Fix It  
The only reliable defense is to **use parameterized queries (prepared statements)**. This ensures user input is treated as data, not executable code.

see [Lab 1: SQL Injection Fundamentals](1.%20SQL%20injection%20vulnerability%20in%20WHERE%20clause%20allowing%20retrieval%20of%20hidden%20data.md).

## Additional Information  
While this lab **requires** the `UNION SELECT` method to return an extra row (as per the objective), another technique called `ORDER BY` can help determine the number of columns in **other SQL injection scenarios**.

### `ORDER BY` Technique (For Future Reference)
- Append `ORDER BY N--` to the parameter, incrementing `N` (1, 2, 3, ...) until an error occurs.
- Example:  
  ```
  category=Pets' ORDER BY 1-- → OK  
  category=Pets' ORDER BY 2-- → OK  
  category=Pets' ORDER BY 3-- → OK  
  category=Pets' ORDER BY 4-- → Error  
  ```
- This suggests the original query returns **3 columns**.

> [!WARNING] **Note:**
>  While `ORDER BY` helps identify column count, it **does not return new data rows** and therefore **cannot solve this specific lab**. However, it's a valuable reconnaissance step in more complex attacks and will be useful in upcoming labs.
